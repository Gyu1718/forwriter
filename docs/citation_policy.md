# Citation Policy

이 문서는 ForWriter Citation Library에서 책 인용을 관리할 때 지켜야 할 원칙을 정리한다.

## 1. 목적

이 저장소는 책 원문을 재배포하기 위한 공간이 아니다. 목적은 글쓰기와 연구를 위한 메타데이터, 직접 작성한 설명, 짧은 인용, 출처 위치, 주제 태그를 관리하는 것이다.

## 2. 저장 가능한 내용

공개 저장소에 저장할 수 있는 정보는 다음으로 제한한다.

- 직접 작성한 책 설명
- 직접 작성한 요약
- 짧은 인용문
- 페이지 또는 위치 정보
- 주제 태그
- 독서 메모
- 활용 메모
- 검수 상태

## 3. 저장하지 않는 내용

다음 자료는 공개 저장소에 저장하지 않는다.

- EPUB 원문 파일
- PDF 원문 파일
- OCR 결과 전체
- 책 전체 텍스트 추출본
- 장 전체 복사본
- 긴 인용 모음
- 출판사의 해설 전문

## 4. 인용 원칙

1. 인용은 필요한 만큼만 짧게 사용한다.
2. 인용은 반드시 출처 위치를 함께 기록한다.
3. 번역본 인용과 원서 인용을 구분한다.
4. AI가 생성한 인용 후보는 원문과 대조하기 전까지 `verified`로 표시하지 않는다.
5. 문맥을 왜곡할 수 있는 단편 인용은 사용하지 않는다.
6. 한 책에서 과도하게 많은 인용을 공개하지 않는다.

## 5. 검수 기준

인용을 `verified`로 바꾸기 전에 다음을 확인한다.

- 원문과 글자가 일치하는가?
- 페이지 또는 위치가 맞는가?
- 문맥상 저자의 의도와 충돌하지 않는가?
- 인용 길이가 과도하지 않은가?
- 출판본, 번역본, 판본 정보가 필요한 경우 기록되어 있는가?

## 6. 상태값 사용

```txt
candidate    자동 추출 또는 초안 단계
needs_check  사람이 원문 대조를 해야 하는 단계
verified     원문·위치·문맥 검수 완료
rejected     사용하지 않기로 결정
```

## 7. 공개와 비공개 분리

원문 파일과 전체 추출 텍스트는 로컬 또는 비공개 저장 공간에 둔다.

```txt
private_sources/
```

이 폴더는 `.gitignore`로 보호한다. 단, 폴더 구조 유지를 위한 `.gitkeep`만 커밋한다.
